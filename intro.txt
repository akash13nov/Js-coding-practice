1. "this" in global scope always points to the globalObject (globalObject depends on javascript runtime env)
2. "this" in a function depends on 2 things
    a. strict / non strict mode (in strict mode, "this" is undefined, in non strict mode, "this substitution" takes place and "this" points to globalObject)
    b. how the function is being invoked (if we invoke the function using the global object in strict mode, then "this" will point to that globalObject)
3. "this" in a method always points to the object that is used to invoke the method.
4. call, apply & bind are used to share a method with another object (hence the "this" keyword reference would also change accordingly)
5. "this" inside arrow function refers to the enclosing lexical context
6. "this" in DOM points to the HTML element itself on which it is being used








1. What is Dependency Injection in Angular? 
- DI is a design pattern where Angular automatically provides dependencies to classes instead of classes creating them manually. Angular has a built-in system that creates services once and then automatically gives these services to any component or other service that requests them in their constructor.

2. What is the @Injectable() decorator used for? 
- @Injectable() marks a class as available for dependency injection, allowing Angular to inject it into other classes.

3. How do you provide a service at the root level? 
- Use @Injectable({ providedIn: 'root' }) or add the service to AppModule's providers array.

4. What is the difference between providing a service in root vs component level? 
- Root level creates a singleton shared across the app, while component level creates a new instance for each component.

5. How do you inject a service into a component? 
- Add the service as a parameter in the component's constructor: constructor(private myService: MyService) {}

6. What is an InjectionToken used for? 
- InjectionToken is used to create tokens for injecting non-class dependencies like strings, objects, or configuration values.

7. How does tree shaking work with Angular services? 
- Services with providedIn: 'root' can be tree-shaken - unused services are automatically removed from the final bundle.

8. What happens when you inject a service outside of injection context? 
- You'll get an error because Angular can't resolve dependencies outside its injection context. Use inject() function to handle this.

9. What is a hierarchical injector in Angular? 
- Angular creates an injector tree where child injectors can access parent injector services, but not vice versa.

10. What is injection context in Angular? 
- Injection context is like a special zone where Angular can provide services. Think of it as Angular's "service delivery area" - you can only get services when you're inside this zone (like inside components or services).

11. What is the difference between tight coupling and loose coupling, and how does DI help? 
Tight coupling means classes directly create their dependencies, making code hard to test. DI creates loose coupling by injecting dependencies, making code more flexible.

    const charCount = {};

       charCount[char] = (charCount[char] || 0) + 1;
    }
