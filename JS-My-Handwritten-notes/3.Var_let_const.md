### let vs var vs const

- var declarations are globally scoped or function scoped while let 
and const are block scoped. 

- var variables can be updated and re-declared within its scope; 
  let variables can be updated but not re-declared; 
  const variables can neither be updated nor re-declared. 


### Notes
- ##### var - Since javascript is came into existence, 
  var variable has been used. This a function scoped/global scoped

- ##### let, const - Since ES6 is came into existence, 
  let variable has been used. This is a block scoped

- ##### let variables cannot be re-declared but <i>can be re-assigned</i>

- ##### const variables cannot be re-declared and re-assigned

- ##### Prefer using const variables unless until you want to change 
  the values
  
  
### My Notes

** var
- var creates a function-scoped/global scoped
- can be updated and re-declared
- Not used in modern JS anymore.

** let
- let creates a block scoped
- cannot be re-declared but <i>can be re-assigned</i>

** const
- const is a constant value.
- const cannot be re-declared and re-assigned.

### What is Block scope
- Block scope in javascript means that a vaiable only exists and
  can be used inside the curly braces {} where it was created.



### Example 1

```js
if (true) {
  var varVariable = 'This is true';
}
console.log(varVariable);

if (true) {
  let letVariable = 'This is true';
  console.log(letVariable);
}

console.log(letVariable);
```

<summary>Solution</summary>

```js
This is true
This is true // inside if condition
Reference Error: 'letVariable' is not defined. // outside
```
