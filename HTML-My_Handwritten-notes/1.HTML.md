# 🌐 HTML (HyperText Markup Language)

- **HTML** is the standard markup language used to create web pages.  

---

## 📌 What is a Markup Language?
- A markup language defines the **structure of a document** using elements like:
  - Headings
  - Paragraphs
  - Links
  - Lists
  - And more...

---

## ⚡ Important Note
- **HTML is not a programming language**, it’s a **markup language** (like XML).




# 🌐 Difference Between HTML and HTML5

---

## 📌 HTML
- Standard markup language used to create web pages.
- Introduced basic structure like headings, paragraphs, tables, forms, images, and links.
- Relies on third-party technologies (like Flash, Silverlight) for multimedia (audio/video).
- Does not support mobile-friendly features by default.
- Limited support for APIs.
- Syntax was stricter (e.g., attributes needed quotes in most cases).

---

## 📌 HTML5
- Latest version of HTML with **new features and APIs**.
- Provides built-in support for **audio** and **video** tags (no need for Flash).
- Introduced **semantic tags** like: - `<header>`, `<footer>`, `<article>`, `<section>`, `<nav>`, etc.
- Supports **graphics and animations** with `<canvas>` and **SVG**.
- Introduced **local storage** and **session storage** (better than cookies).
- Provides **Geolocation API**, **Drag & Drop API**, **Web Workers**, etc.
- Mobile-friendly and responsive design support.
- Supports offline applications via **Application Cache** and **IndexedDB**.
  
  
# ❓ What if you remove `<!DOCTYPE html>` from your HTML?

## ✅ Answer:
- The `<!DOCTYPE html>` declaration tells the browser that we are using **HTML5**.  
- If the document type is **not mentioned**, the browser will go into **Quirks Mode** (also called Compatibility Mode).  
- **Quirks Mode** behavior depends on the web browser version:
- In older browsers, HTML5 tags like `<header>`, `<footer>`, `<section>`, etc., will **not be supported**.
- Modern browsers may still try to render the page, but it can lead to **inconsistent styling and layout issues**.  


### What are Semantic Elements in HTML?
Ans : Semantic elements in HTML are elements that clearly describe their meaning and purpose both to the browser and to developers.

#🔹 Examples of Semantic Elements
   **<header>**   – Represents the header section of a webpage or section.
	**<footer>**   – Represents the footer section of a webpage or section.
	**<article>**  – Represents a self-contained piece of content.
	**<section>**  – Represents a thematic grouping of content.
	**<nav>**      – Represents a section of navigation links.
	**<aside>**    – Represents content indirectly related to the main content (like a sidebar).


## 10 more semantic elements:

1. **`<progress>`** – Displays the progress of a task.  
2. **`<nav>`** – Contains navigation links for a webpage.  
3. **`<time>`** – Represents a specific period in time or a date.  
4. **`<mark>`** – Highlights parts of the text for reference or emphasis.  
5. **`<summary>`** – Provides a summary, caption, or legend for a `<details>` element.  
6. **`<meter>`** – Represents a scalar measurement within a known range (like a gauge).  
7. **`<figure>`** – Contains content like images, illustrations, diagrams, etc., along with a caption.  
8. **`<details>`** – Represents additional information or controls that can be toggled open or closed.  
9. **`<aside>`** – Contains content that is related to the main content but can be considered separate.  
10. **`<article>`** – Represents a self-contained composition in a document, such as a blog post or a news article.
